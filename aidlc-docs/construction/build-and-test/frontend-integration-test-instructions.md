# Frontend Integration Test Instructions

## 목적

Frontend ↔ Backend API 연동이 정상 동작하는지 검증합니다.

## 사전 조건

```bash
# Backend + MySQL 실행 필수
docker-compose up -d mysql backend

# Frontend 실행
cd frontend && npm run dev
```

브라우저에서 `http://localhost:5173` 접속

## 테스트 시나리오

### 시나리오 1: 고객 주문 플로우

| 단계 | 동작 | 기대 결과 |
|------|------|----------|
| 1 | `/table/setup` 접속 → 테이블 번호 입력 | 메뉴 페이지로 이동 |
| 2 | 카테고리 사이드바에서 카테고리 선택 | 해당 카테고리 메뉴 표시 |
| 3 | 메뉴 카드 클릭 → 수량 선택 → 장바구니 추가 | 장바구니에 항목 추가됨 |
| 4 | 장바구니 페이지 → 주문하기 | 주문 성공 메시지, 주문내역 페이지 이동 |
| 5 | 주문내역 페이지에서 주문 확인 | 주문 상태 "대기중" 표시 |

### 시나리오 2: 관리자 로그인 + 주문 관리

| 단계 | 동작 | 기대 결과 |
|------|------|----------|
| 1 | `/admin/login` → 관리자 로그인 | 대시보드로 이동 |
| 2 | 테이블 관리 → 테이블 목록 확인 | 테이블 목록 + 주문 현황 표시 |
| 3 | 주문 상세 → 주문 접수 | 주문 상태 "준비중"으로 변경 |
| 4 | 주문 완료 처리 | 주문 상태 "완료"로 변경 |

### 시나리오 3: 관리자 메뉴/카테고리 관리

| 단계 | 동작 | 기대 결과 |
|------|------|----------|
| 1 | 메뉴 관리 → 메뉴 추가 | 폼 다이얼로그 → 메뉴 생성 성공 |
| 2 | 메뉴 수정 | 수정 폼 → 저장 성공 |
| 3 | 카테고리 관리 → 카테고리 추가 | 카테고리 생성 성공 |
| 4 | 이미지 업로드 | 이미지 미리보기 + 업로드 성공 |

### 시나리오 4: SSE 실시간 알림

| 단계 | 동작 | 기대 결과 |
|------|------|----------|
| 1 | 관리자 대시보드 열기 | SSE 연결 수립 |
| 2 | 다른 탭에서 고객 주문 생성 | 관리자 화면에 실시간 알림 표시 |
| 3 | 서비스 호출 버튼 클릭 | 관리자 화면에 서비스 호출 알림 |

### 시나리오 5: 테이블 정산

| 단계 | 동작 | 기대 결과 |
|------|------|----------|
| 1 | 관리자 → 테이블 선택 → 정산 | 정산 다이얼로그 (총액 표시) |
| 2 | 정산 확인 | 테이블 상태 초기화, 주문 정산 완료 |
