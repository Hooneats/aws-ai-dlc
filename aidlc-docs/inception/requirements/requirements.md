# 테이블오더 서비스 - 요구사항 정의서

## Intent Analysis

- **User Request**: 테이블오더 서비스 구축 (디지털 주문 시스템)
- **Request Type**: New Project (Greenfield)
- **Scope**: Multiple Components (고객 UI, 관리자 UI, Backend API, Database)
- **Complexity**: Complex (실시간 통신, 세션 관리, 다중 사용자 유형, CRUD 전반)

---

## 1. 기술 스택

| 영역 | 기술 |
|------|------|
| Backend | NestJS (TypeScript) |
| Frontend | React (TypeScript) |
| Database | MySQL |
| 실시간 통신 | Server-Sent Events (SSE) |
| 인증 | JWT 토큰 |
| 배포 환경 | 로컬 개발 환경 (Docker Compose) |
| 이미지 저장 | 로컬 서버 파일 업로드 |

---

## 2. 아키텍처 결정사항

- **Frontend 구조**: 단일 React 프로젝트에서 라우팅으로 고객용/관리자용 분리
- **매장 구조**: 단일 매장만 지원 (MVP 단순화)
- **관리자 계정**: Seed 데이터로 초기 생성 (회원가입 기능 없음)
- **고객 실시간 업데이트**: SSE로 주문 상태 실시간 반영 포함

---

## 3. 기능 요구사항

### 3.1 고객용 기능 (Customer)

#### FR-C01: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 초기 설정 1회 수행 (매장 식별자, 테이블 번호, 테이블 비밀번호)
- 로그인 정보 로컬 저장 후 자동 로그인
- 테이블 세션 추적용 세션 ID 관리

#### FR-C02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본 화면으로 표시
- **추천 메뉴 섹션이 메뉴 화면 상단에 표시** (관리자가 지정한 메뉴)
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지
- **할인 적용 메뉴: 할인율, 원래 가격(취소선), 할인된 가격 표시**
- **품절 메뉴: 품절 표시 및 주문 불가 처리**
- 카테고리 간 빠른 이동
- 카드 형태 레이아웃, 터치 친화적 버튼 (최소 44x44px)

#### FR-C03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 로컬 저장 (페이지 새로고침 시 유지)
- 서버 전송은 주문 확정 시에만 수행

#### FR-C04: 주문 생성
- 주문 내역 최종 확인 후 주문 확정
- **주문 시 요청사항 입력 가능** (예: "덜 맵게", "얼음 빼주세요" 등, 선택 입력)
- 주문 성공 시: 주문 번호 표시, 장바구니 자동 비우기, 메뉴 화면으로 자동 리다이렉트
- 주문 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 식별, 테이블 식별, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID, 요청사항

#### FR-C05: 주문 내역 조회

#### FR-C06: 서비스 호출
- 고객이 테이블에서 직원 호출, 물, 수저 등 서비스 요청을 할 수 있는 기능
- 서비스 호출 버튼을 통해 요청 (직원 호출, 물, 수저, 앞접시, 냅킨 등)
- 요청 시 관리자 대시보드에 SSE로 실시간 알림
- 관리자가 요청 확인/완료 처리 가능
- 관리자가 서비스 호출 항목을 관리할 수 있음 (추가/수정/삭제)

#### FR-C05: 주문 내역 조회
- 주문 시간 순 정렬
- 주문별 상세: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)
- SSE를 통한 주문 상태 실시간 업데이트
- 현재 테이블 세션 주문만 표시 (이전 세션 제외)

### 3.2 관리자용 기능 (Admin)

#### FR-A01: 매장 인증
- 매장 식별자, 사용자명, 비밀번호 입력
- JWT 토큰 기반 인증, 16시간 세션 유지
- 브라우저 새로고침 시 세션 유지, 16시간 후 자동 로그아웃
- 비밀번호 bcrypt 해싱, 로그인 시도 제한

#### FR-A02: 실시간 주문 모니터링
- SSE 기반 주문 목록 실시간 업데이트
- 그리드/대시보드 레이아웃: 테이블별 카드 형태
- 각 테이블 카드: 총 주문액, 최신 주문 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기 (요청사항 포함)
- 주문 상태 변경 (대기중 → 준비중 → 완료)
- 신규 주문 시각적 강조
- **신규 주문/서비스 호출 시 브라우저 소리 알림** (알림음 on/off 토글)
- 2초 이내 주문 표시

#### FR-A03: 테이블 관리
- **테이블 추가/삭제** (매장 좌석 변경 시 테이블 수 조정)
- 테이블 태블릿 초기 설정 (테이블 번호, 비밀번호, 16시간 세션)
- 주문 삭제 (확인 팝업, 즉시 삭제, 총 주문액 재계산)
- **테이블 정산** (이용 완료): 관리자가 정산 버튼을 눌러 테이블 세션 종료. 주문 내역을 과거 이력으로 이동, 현재 주문/총액 리셋, 다음 손님이 새 세션으로 이용 가능
- 과거 주문 내역 조회: 테이블별 과거 주문 목록 (시간 역순), 날짜 필터링

#### FR-A04: 메뉴 관리
- 메뉴 CRUD (조회, 등록, 수정, 삭제)
- 메뉴 정보: 메뉴명, 가격, 설명, 카테고리, 이미지 (로컬 서버 업로드)
- 메뉴 노출 순서 조정
- **메뉴 품절 설정/해제** (품절 시 고객 화면에서 주문 불가 처리)
- **추천 메뉴 지정/해제** (선택한 메뉴를 고객 화면 상단에 추천 메뉴로 노출)
- **메뉴별 할인 설정** (할인율(%) 지정/해제, 할인 적용 시 고객 화면에 할인율과 할인가 자동 표시)
- 필수 필드 검증, 가격 범위 검증

#### FR-A04-1: 카테고리 관리
- 카테고리 CRUD (조회, 추가, 수정, 삭제)
- 카테고리 노출 순서 조정
- 카테고리 삭제 시 해당 카테고리 메뉴 처리 (다른 카테고리로 이동 또는 삭제 불가)

#### FR-A05: 서비스 호출 관리
- 고객의 서비스 호출 요청을 SSE로 실시간 수신 (테이블 번호, 요청 항목 표시)
- 요청 확인/완료 처리
- 서비스 호출 항목 관리 (직원 호출, 물, 수저, 앞접시, 냅킨 등 추가/수정/삭제)

#### FR-A06: 매출 통계
- **일별 매출 요약**: 선택한 날짜의 총 매출, 총 주문 건수
- **테이블별 매출**: 각 테이블의 매출 합계, 주문 건수
- **메뉴별 판매 현황**: 메뉴별 판매 수량, 매출액 (판매량 순 정렬)
- **날짜 선택**: 특정 날짜의 통계 조회 (기본값: 당일)

## 4. 비기능 요구사항

### NFR-01: 성능
- SSE 기반 실시간 통신: 주문 발생 후 2초 이내 관리자 화면 표시
- 장바구니 로컬 저장으로 서버 부하 최소화

### NFR-02: 보안
- JWT 토큰 기반 인증 (관리자 16시간, 테이블 16시간)
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한

### NFR-03: 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 직관적인 카드 형태 레이아웃
- 명확한 시각적 계층 구조

### NFR-04: 배포
- Docker Compose 기반 로컬 개발 환경
- Backend, Frontend, MySQL 컨테이너 구성

- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (~~데이터 분석, 매출 리포트~~, 직원 관리, 예약, 리뷰, 다국어) ※ 일별 매출 통계는 FR-A06으로 MVP에 포함
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어, 지도/번역 API)

## 6. 데이터 모델 (개요)

### 주요 엔티티
- **Store**: 매장 정보 (단일 매장)
- **Admin**: 관리자 계정 (Seed 데이터)
- **Table**: 테이블 정보 (번호, 비밀번호, 세션 ID)
- **Category**: 메뉴 카테고리
- **Menu**: 메뉴 항목 (이름, 가격, 설명, 이미지, 카테고리, 노출 순서, 추천 여부, 할인율, 품절 여부)
- **Order**: 주문 (테이블, 세션 ID, 상태, 총 금액, 주문 시각, 요청사항)
- **OrderItem**: 주문 항목 (메뉴, 수량, 단가)
- **OrderHistory**: 과거 주문 이력 (세션 종료 시 이동)
- **ServiceCallItem**: 서비스 호출 항목 정의 (직원 호출, 물, 수저, 앞접시, 냅킨 등)
- **ServiceCall**: 서비스 호출 요청 (테이블, 요청 항목, 상태, 요청 시각)
