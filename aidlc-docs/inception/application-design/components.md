# 컴포넌트 정의

## Backend 컴포넌트 (NestJS Modules)

### 1. AuthModule
- **목적**: 인증 및 권한 관리
- **책임**: 관리자 로그인(JWT 발급, 16시간), 테이블 태블릿 인증(자동 로그인), bcrypt 해싱, 로그인 시도 제한, JWT Guard(관리자용/테이블용)

### 2. StoreModule
- **목적**: 매장 정보 관리
- **책임**: 매장 기본 정보(단일 매장), Seed 데이터 초기화(매장, 관리자 계정)

### 3. TableModule
- **목적**: 테이블 관리
- **책임**: 테이블 CRUD(추가/삭제), 초기 설정(번호, 비밀번호 없음), 세션 관리(시작/종료), 정산(모든 주문 완료/취소 확인→이력 이동/리셋)

### 4. CategoryModule
- **목적**: 메뉴 카테고리 관리
- **책임**: 카테고리 CRUD, 노출 순서 관리, 삭제 시 메뉴 "기타" 카테고리로 이동, "서비스 요청" 카테고리 지원, "기타" 카테고리(관리자 전용, 고객 미노출)

### 5. MenuModule
- **목적**: 메뉴 관리
- **책임**: 메뉴 CRUD(Soft Delete), 이미지 업로드(로컬), 추천/할인/품절 설정, 노출 순서 관리

### 6. OrderModule
- **목적**: 주문 처리
- **책임**: 주문 생성(품절 검증/할인가 적용/요청사항), 상태 변경(대기중→준비중→완료, 취소), 주문 삭제(전체/항목 단위), 내역 조회(현재 세션/과거), 정산 시 이력 이동

### 7. ServiceCallModule
- **목적**: 서비스 호출 요청 처리
- **책임**: 서비스 호출 요청 생성(카테고리 내 메뉴 항목 기반), 확인/완료 처리, 요청 목록 조회

### 8. SseModule
- **목적**: 실시간 통신 (Server-Sent Events)
- **책임**: SSE 연결 관리(관리자/고객), 신규 주문 이벤트, 주문 상태 변경 이벤트, 서비스 호출 이벤트

### 9. StatisticsModule
- **목적**: 매출 통계 및 운영 대시보드
- **책임**: 실시간 현황 대시보드(오늘 매출/주문/영업테이블/객단가/인기메뉴TOP5/시간대별추이), 일별 매출 통계, 기간별 매출 통계(최대 1년, 추이/순위/요일패턴)

### 10. UploadModule
- **목적**: 파일 업로드
- **책임**: 이미지 파일 업로드(로컬 저장), 정적 파일 서빙

## Frontend 컴포넌트 (React)

### 고객용 페이지
- **MenuPage**: 메뉴 조회 (추천 + 카테고리별 + 서비스 요청 카테고리 + 할인/품절)
- **CartPage**: 장바구니 (추가/수량/삭제/총 금액)
- **OrderPage**: 주문 확정 (최종 확인 + 요청사항)
- **OrderHistoryPage**: 주문 내역 (현재 세션, SSE 실시간 상태)
- **TableSetupPage**: 테이블 초기 설정

### 관리자용 페이지
- **LoginPage**: 관리자 로그인
- **DashboardPage**: 실시간 주문/서비스 호출 모니터링 (그리드 + SSE + 소리 알림)
- **TableManagementPage**: 테이블 관리 (추가/삭제/설정/정산/과거 내역)
- **MenuManagementPage**: 메뉴 관리 (CRUD + 추천/할인/품절 + 서비스 항목)
- **CategoryManagementPage**: 카테고리 관리 (CRUD + 순서)
- **StatisticsPage**: 매출 통계

### 공통 Provider
- **AuthProvider**: 인증 상태 (JWT 저장/갱신/만료)
- **SseProvider**: SSE 연결 (자동 재연결)
- **CartProvider**: 장바구니 상태 (로컬 저장)
